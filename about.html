ReMarkdown about styles config

You should use ReMarkdown!

Is your HTML content looking all HTML-y and CSS-y? I bet it has great typography and carefully crafted negative space?

Such ennui! Ditch all this @font-face and font-size nonsense, and go with a good old plain text look instead!

That’s what ReMarkdown does: it takes your HTML and paints it so that it looks like Markdown. Pretty nerdy, uh? By the way, the text you’re reading right now is not plain text, it’s good old HTML with H1s and Ps and BLOCKQUOTEs etc. Just do “Inspect Element” and you’ll see.

Using ReMarkdown is as simple as downloading remarkdown.css, and declaring it like this: 

<!doctype html>
<html lang="en">
<head>
	<title>Using ReMarkdown</title>
	<link rel="stylesheet" href="remarkdown.css">
</head>
<body data-rmd>
	<h1>Hello World</h1>
	<p>A paragraph.</p>
</body>
</html>

There’s a bunch of alternative styles. You can define your prefered styles using the data-rmd attribute, or when compiling from the Sass source. See the configuration guide for details.

Technicalities

ReMarkdown is on Github, report bugs there if any. The DO WHAT THE FUCK YOU WANT TO PUBLIC LICENSE applies.

Browser support

I’m targetting the most current browsers (mid-2014). Theoretically it should work on IE9+, but that’s a very much untested assertion.

I’m keeping around a legacy ReMarkdown 1.0. Released in 2011, it had partial IE8 support and used a different syntax. It will not be updated.

Not a converter

ReMarkdown is a stylesheet, not an HTML-to-Markdown converter. If you’re looking for one, there’s pandoc for instance.
