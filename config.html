HTML options for alternate styles

You can define a number of options by using keywords in the data-rmd attribute. Keywords must be separated by whitespace. (Think of the data-rmd attribute as a class attribute that is specific to ReMarkdown.) 

<div data-rmd="h1-underline em-underscore hr-center">
	<!-- 
		Any basic HTML content:
		headings, paragraphs, lists, etc.
	-->
</div>

Here is a list of all available styles. Note that if you use the default ReMarkdown build, all the styles specified as “Default” are already active, so you don’t need to add them to the value of the data-rmd attribute.

Style name Default? Description
hn-hash Yes Add hashes at the start of titles
hn-close — Add hashes at the end of titles
h1-underline — Underline H1s with equals
h2-underline — Underline H2s with hyphens
ul-hyphen Yes Use “-” for bullets
ul-star — Use “*” for bullets
ul-plus — Use “+” for bullets
em-star Yes Use “*” signs for emphasis
em-underscore — Use “_” signs for emphasis
strong-star Yes Use “*” signs for strong emphasis
strong-underscore — Use “_” signs for strong emphasis
a-bracket Yes Add square brackets around links.
a-showurl — Show URLs after links
hr-center — Horizontally center the HR
del-gfm — GitHub-flavored Markdown style for DEL elements

Build a custom stylesheet with Sass

You can recompile ReMarkdown using Sass, and there are a number of options (Sass variables) that you can use to alter the CSS output of your custom build. Use this template which has documentation for all the available options. Some highlights:

Change the default styles

You can use the $defaults variable to change the styles that should be applied by default (i.e. when you use the data-rmd attribute without any explicit style). 

$defaults:
	hn-hash h1-underline h2-underline
	ul-star em-underscore strong-underscore
	hr-hyphen hr-center;

You will need to list all the styles you want to see apply by default. That can be a fairly long list (as above). For instance, if I omit both em-star and em-underscore, EM elements won’t have any visible marker (unless we explicitly set one of those styles in the HTML code).

Do not include alternate styles

By setting the $output_all_styles option to false, ReMarkdown will only output those styles that are listed in $defaults. This can be useful if you want to pick a set of default styles, don’t intend to ever declare the alternative styles for each element, and want to make a smaller CSS file.

Currently the default build with all styles included is close to 7.5 KB, while the same defaults with no alternative styles included is down to 4.0 KB.
